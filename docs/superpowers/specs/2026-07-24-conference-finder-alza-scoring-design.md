# conference-finder "알짜 점수" 도입 설계

- 날짜: 2026-07-24
- 대상 스킬: `plugins/conference-finder/skills/conference-finder/SKILL.md`
- 목적: 주제 관련도(relevance)만으로 걸러지던 추천을, **실제로 값진(깊이·사람) 이벤트**를 앞세우도록 개선한다.

## 배경 / 문제

현재 스킬은 관심 프로필 키워드에 맞는지("관련도")만 본다. 그래서 알맹이 있는 딥다이브와 홍보성 데모데이가 동일하게 상위에 오른다. 사용자가 원하는 "알짜배기"의 정의는 두 축:

1. **깊이** — 배울 게 많은 자리 (실습·딥다이브·발표 밀도)
2. **사람** — 만날 사람이 좋은 자리 (빌더·현업·선별된 크라우드)

둘 다 **제목엔 안 드러나고**, 이벤트의 실제 데이터를 봐야 판별된다.

## 데이터 가용성 검증 결과 (2026-07-24, 실측)

Luma `get-paginated-events` 리스트 API가 이미 다음 필드를 구조화해 제공함(추가 fetch 불필요):

| 신호 | 필드 | 축 |
|------|------|-----|
| 선별·심사제 | `ticket_info.require_approval` | 사람 |
| 주최자 정체/검증 | `calendar.name`, `calendar.is_personal`, `calendar.verified_at`, `calendar.website` | 사람 |
| 규모·주최 인원 | `hosts[]`, `guest_count`, `featured_guests` | 사람 |
| 유·무료(진정성 프록시) | `ticket_info.is_free` | 보조 |

안 잡히는 것: **형식/아젠다(깊이 축)** — 리스트 API에 이벤트 설명 본문이 없음.

- 개별 JSON 엔드포인트 `api.luma.com/url/get?url=<slug>` → **404**(사용 불가, 추측 금지).
- 개별 이벤트 **HTML 페이지 `luma.com/<slug>` WebFetch → 설명 본문·형식 분류 정상 추출**(실측: "강연 아닌 커뮤니티 모임"까지 정확히 분류). 날짜 등 구조화 메타는 여전히 JS라 불안정하지만, 설명 프로즈는 SEO용으로 HTML에 있어 잡힘.

→ **사람 축은 API JSON에서 공짜, 깊이 축은 상·중 후보만 개별 페이지 설명 정독**으로 얻는다.

## 채점 루브릭 — 새 축 `⭐알짜` (관련도와 별개)

깊이·사람 각 0~3점, 합산 `⭐0~6`. **근거가 확인 안 되면 그 항목 0점(감점 아님), 지어내지 않는다.**

### 깊이 점수 (0~3) — 출처: 개별 페이지 설명 정독
- 형식: 핸즈온/워크숍/코드랩/딥다이브/포스트모템 `+` · 데모데이/피칭/쇼케이스/단순홍보 `−`
- 아젠다 밀도: 세션·트랙 여럿 `+` · 키노트 하나+친목 `−`
- 발표자 이력: **페이지에 명시된 직함·소속만** 반영. 이름별 외부 조회는 하지 않는다(비용·날조 방지).

### 사람 점수 (0~3) — 출처: 리스트 API JSON
- 선별성: `require_approval=true`(지원·심사·초대) `+`
- 주최자 신뢰: `calendar.verified_at` 있음 또는 조직 캘린더(`is_personal=false`)+`website` `+` · 개인(`is_personal=true`) 캘린더 `0`
- 규모·성격: `hosts`가 복수/현업 소속, `guest_count`가 밀도 있는 빌더 모임 `+` · 구직·홍보성 `−`

## 흐름 통합 — 깔때기

기존 Step 4(관련도 순위화) 뒤에 삽입:

- **Step 4.7a — JSON 알짜 신호 채점(전 후보, 공짜)**: 리스트 API 필드로 사람 축 점수 부여.
- **Step 4.7b — 설명 정독(깊이 축, 관련도 상·중만)**: 개별 페이지 `WebFetch`로 형식/아젠다 확인해 깊이 축 점수 부여. 관련도 `하`는 정독 생략(깊이 0, 표기 "정독 생략").
  - **thin-page 폴백**: 설명이 비면 제목 키워드로만 대략 잡고 "정독했으나 형식 불명"으로 명시 — 조용한 0점 금지.
- **Step 4.8 — 평판 확인(C, ⭐ 상위 3~5개만)**: `WebSearch`로 과거 회차 후기·주최자 트랙레코드를 찾아 `⭐` ±보정.
- **정렬 키 변경**: 1순위 `⭐알짜` 내림차순, 2순위 관련도. (관심 프로필은 통과선 유지.)

## 출력·Notion 반영

- **한눈 표**: `| # | 이벤트 | 유형 | 날짜 | 관련도 | ⭐알짜 |`
- **상세 카드**: `알짜 근거` 한 줄 추가 — 확인된 신호로("지원제·주최 AIB(검증)·실습형" 식). 근거 없으면 "정독했으나 알짜 신호 약함".
- **Notion 컨퍼런스 트래커**: `알짜`(⭐0~6 또는 상/중/하) 속성 추가, 근거는 `요약`에 이어 붙임(별도 컬럼 신설은 선택).

## 비용

관련도 상·중이 보통 10개 안팎이라 깊이 정독 WebFetch 감당 가능. 사람 축은 fetch 0, 평판은 상위 3~5개만 → 비용 폭발 없음.

## 범위 밖 (YAGNI)

- 발표자 개인별 외부 이력 조회(OSS 스타 수 등) — 하지 않음.
- Meetup/웹검색 소스에 대한 알짜 채점 — 1차는 Luma 필드 기반. Meetup은 형식 키워드만 반영(깊이 축 부분), 사람 축은 데이터 없으면 0.
