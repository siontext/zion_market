# 컨퍼런스 후기 LinkedIn 게시글 작성

다녀온 컨퍼런스의 URL을 받아 내용을 파악하고, 사용자의 개인 소감을 더해 LinkedIn에 올릴 후기 게시글을 작성한다.

## 진행 순서

체크리스트로 만들어 순서대로 진행한다.

### 1. 컨퍼런스 URL 확보

- 사용자가 URL을 이미 제공했으면 그것을 사용한다.
- 없으면 "다녀오신 컨퍼런스 페이지 URL을 알려주세요" 라고 한 번 물어본다.

### 2. 컨퍼런스 정보 파악

- `WebFetch`로 해당 URL을 읽어 다음을 추출한다:
  - 컨퍼런스 이름
  - 개최 일자·장소
  - 주제 / 핵심 키워드
  - 주요 세션·연사·트랙
- 페이지 정보가 부족하면 `WebSearch`로 보완한다.
- 파악한 내용을 **3~5줄로 요약해서 사용자에게 먼저 보여주고**, "이 컨퍼런스가 맞나요?" 라고 확인한다.

### 3. 개인 소감 수집 (간단히)

확인이 되면 아래를 **한 번에 간단히** 물어본다.

- 가장 인상 깊었던 세션 / 배운 점 1~2가지
- (선택) 함께한 사람, 느낀 점, 앞으로 적용해보고 싶은 것

**사용자가 소감을 건너뛰거나 "알아서 해줘"라고 하면**: 컨퍼런스 정보(주제·주요 세션)를 근거로 **인사이트 후보 2~3개를 제안**하고, "이 중 실제로 와닿았던 게 있나요? 골라주시면 그 방향으로 쓸게요"라고 되묻는다.
- 제안은 어디까지나 후보다. 사용자가 고르거나 승인한 것만 게시글에 반영한다.
- 끝내 아무 선택도 없으면, 해당 인사이트 문단은 자연스러운 자리표시자(예: `[여기에 가장 인상 깊었던 세션 소감을 채워주세요]`)로 남기고 나머지는 완성한다. 사용자의 감상을 사실인 것처럼 지어내지 않는다.

### 4. 게시글 초안 작성

**언어·톤**: 한국어, 전문적이면서 친근한 링크드인 톤. 과장·클리셰("정말 유익했습니다" 남발) 지양, 구체적인 내용 우선.

**구조**:
1. **훅 (1~2줄)** — 참석 사실과 한 줄 소감으로 시선을 끈다.
2. **컨퍼런스 소개** — 이름·일자·주제를 자연스럽게 녹인다.
3. **인상 깊었던 점 / 인사이트** — 사용자가 준 소감을 중심으로 구체적으로. 세션·연사 이름을 언급하면 신뢰도가 올라간다.
4. **마무리 소감** — 배운 점을 어떻게 적용할지, 또는 감사 인사.
5. **해시태그** — 컨퍼런스명·주제 관련 5~8개 (한국어/영어 혼용 가능).

**분량**: LinkedIn 기준 800~1500자 내외. 문단은 짧게, 줄바꿈을 활용해 모바일에서 읽기 좋게 한다.

### 5. 초안 제시 및 수정

- 완성된 게시글을 코드 블록이 아닌 일반 텍스트로 보여준다(그대로 복사해 올릴 수 있게).
- "톤·길이·강조할 부분 등 수정할 점이 있으면 말씀해주세요"라고 안내하고, 요청에 따라 다듬는다.

## 주의사항

- 컨퍼런스 정보를 지어내지 않는다. URL·검색으로 확인된 사실만 쓰고, 불확실하면 사용자에게 묻는다.
- 사용자가 주지 않은 개인적 감상·인용을 임의로 만들지 않는다. 인사이트를 자동 제안할 때도 사용자가 고르거나 승인한 것만 반영하며, 사용자의 경험인 것처럼 단정해서 쓰지 않는다.
