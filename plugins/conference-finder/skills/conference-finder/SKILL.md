# 갈만한 컨퍼런스·밋업 찾기

Luma와 Meetup에서 다가오는 이벤트를 모아, 사용자의 관심 프로필에 맞는 것을 관련도순으로 추천한다.

## 관심 프로필 (기본값)

아래를 관련도 평가의 기본 기준으로 삼는다. 사용자가 실행 시 "이번엔 백엔드만", "무료만", "주말만" 식으로 좁히면 그 조건을 우선한다.

- **AI** (전반)
- **AI 에이전트 / LLM 애플리케이션**
- **백엔드 개발**
- **신기술**
- **개발자 네트워킹**
- **스타트업 / 빌더 / 해커톤**

"가서 인사이트를 많이 얻을 수 있는" 자리를 선호한다 → 단순 홍보성 행사보다, 실무 발표·핸즈온 워크숍·깊이 있는 네트워킹이 있는 이벤트에 가산점을 준다.

## 진행 순서

체크리스트로 만들어 순서대로 진행한다.

### 1. Luma 서울 목록 읽기 (1차 소스)

- `WebFetch`로 `https://luma.com/seoul` 을 읽어 다가오는 이벤트 목록을 추출한다.
  - 각 이벤트: 제목, 장소, 주최자, 상태(매진·대기 등). (날짜는 목록에서 안 나올 수 있음 — 그래도 진행)
- 다른 도시를 요청하면 `https://luma.com/<city>` 로 바꾼다 (예: `luma.com/busan`).

### 2. Meetup 검색 읽기 (2차 소스)

- 관심 키워드별로 `WebFetch`로 Meetup 검색 페이지를 읽는다. URL 형식:
  `https://www.meetup.com/find/?keywords=<키워드>&location=kr--11--Seoul&source=EVENTS`
- 키워드는 관심 프로필에서 1~3개를 골라 각각 조회한다 (예: `AI`, `backend`, `DevOps`). Meetup은 날짜가 잘 나오므로 날짜 정보를 여기서 확보한다.

### 3. 관련도 평가·순위화 (중복 제거)

- Luma·Meetup 결과를 합치고, 같은 이벤트는 하나로 합친다.
- 각 이벤트를 관심 프로필과 대조해 **관련도(상/중/하)**를 매기고, "인사이트 가치"가 높아 보이는 순으로 정렬한다.
- 사용자가 좁힌 조건(주제·무료·요일·지역)이 있으면 그걸로 먼저 거른다.

### 4. WebSearch 보강 (3차, 선택)

- Luma·Meetup에 안 잡히는 큰 컨퍼런스가 있을 수 있으므로, 여유가 되면 `WebSearch`로 보강한다 (예: "2026 서울 AI 컨퍼런스").
- WebSearch는 US 기준 결과라 한국 이벤트엔 약할 수 있다. **보조로만** 쓰고, 결과가 부실하면 조용히 넘어간다.

### 4.5. 상위 추천의 날짜 백필 (Luma 대응)

- Luma 서울 목록은 날짜가 안 나오는 경우가 많다. **최종 제시할 상위 추천(5~8개) 중 날짜가 비어 있는 이벤트**는, 개별 이벤트 페이지를 `WebFetch`로 열어 **날짜·시간을 채운 뒤** 제시한다.
  - 개별 URL은 `luma.com/<slug>` 형식. `lu.ma/<slug>`는 `luma.com/<slug>`로 리다이렉트되므로 처음부터 `luma.com`으로 요청한다.
  - 개별 슬러그를 못 찾았으면 날짜를 지어내지 말고 "목록에서 날짜 미확인"으로 표기한다.
- Meetup은 목록에서 날짜가 잘 나오므로 백필이 대개 불필요하다.

### 5. 추천 목록 제시

- 상위 5~8개를 아래 형식으로 보여준다:
  - **제목** — 관련도(상/중/하), 날짜(있으면), 장소, 주최자, 상태
  - **출처**: 어느 플랫폼에서 찾았는지(Luma / Meetup / 웹검색)와 **링크를 함께 표기**한다.
    - 개별 이벤트 URL을 확보했으면 그 URL을 쓴다.
    - 개별 URL이 목록에서 안 잡혔으면(Luma 서울 페이지 등), 최소한 **찾은 출처 목록 페이지 URL**(예: `https://luma.com/seoul`, `https://www.meetup.com/find/?keywords=...`)을 표기하고 "개별 링크는 상세 확인 단계에서 찾는다"고 밝힌다.
  - **추천 이유** 한 줄 (어떤 관심사에 맞는지, 왜 인사이트가 있을지)
- 목록 끝에 "더 자세히 보고 싶은 이벤트를 골라주세요"라고 안내한다.

### 6. 상세 확인

- 사용자가 고른 이벤트의 **개별 페이지 URL**을 `WebFetch`로 열어 정확한 날짜·시간·장소·신청 링크·상세 내용을 확인해 알려준다.
- 개별 페이지 URL을 모르면, 제목으로 `WebSearch`하거나 사용자에게 링크를 물어본다.

## 주의사항

- 이벤트 정보를 지어내지 않는다. WebFetch·WebSearch로 확인된 것만 쓰고, 날짜·링크가 불확실하면 "목록에서 날짜가 확인되지 않음"이라고 솔직히 밝힌다.
- 매진·마감된 이벤트는 상태를 함께 표기해 사용자가 헛걸음하지 않게 한다.
- 다녀온 뒤 후기를 쓰고 싶어 하면 `conference-recap` 스킬로 이어준다.
